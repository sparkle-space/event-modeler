Element Gallery

The six core element types used on the EventModeler canvas. Each serves a distinct role in the event model.

C
Command
An intent to change state. Commands are validated, then either accepted (producing an event) or rejected (producing an exception).
E
Event
An immutable fact that something happened. Events are the single source of truth in the system and drive all downstream projections.
V
View
A read model or projection built from one or more events. Views provide the data users and systems query for display or decisions.
T
Trigger
A user action or external signal that initiates a command. Triggers represent the starting point of a workflow slice.
!
Exception
A failure or rejection path. Exceptions represent what happens when a command cannot be fulfilled due to business rule violations.
A
Automation
A process manager or policy that reacts to events and issues commands automatically. Automations connect slices into larger workflows.
